# AI宠物形象重建Demo - 产品需求文档

## 概述
- **摘要**: 构建一个完整的AI宠物形象重建Demo系统，用户可通过上传宠物照片/视频，由AI生成具有眨眼等动态效果的2D数字宠物形象，并支持桌面宠物模式的交互展示。
- **目的**: 验证"宠智灵境"平台核心AI能力的可行性，为用户提供可视化的宠物数字复刻体验，作为产品MVP的核心功能模块。
- **目标用户**: 失去宠物的主人、想要为现有宠物创建数字形象的用户、对AI图像生成技术感兴趣的早期体验者。

## 项目目标
1. 实现完整的图片/视频上传与预处理流程（支持多文件上传、预览、格式验证）
2. 集成AI图像生成能力（支持调用外部API或本地模型），将宠物照片转换为2D卡通风格形象
3. 实现2D动态宠物渲染系统，包含眨眼、呼吸、轻微晃动等自然动画效果
4. 构建桌面宠物模式UI，支持拖拽、点击交互、窗口置顶等功能
5. 提供完整的用户操作流程：上传→生成→预览→保存→桌面展示

## 非目标（不在本阶段范围）
- 不实现完整的3D宠物建模和骨骼动画系统
- 不实现语音合成和实时对话功能
- 不实现云端数据持久化和用户账户系统
- 不实现多平台原生应用打包（Electron/Tauri）
- 不实现复杂的AI训练和模型微调功能

## 背景与技术约束
- **现有技术栈**: Next.js 14 + React 18 + Tailwind CSS 3.4
- **项目定位**: "宠智灵境"平台的前端Demo原型
- **当前状态**: 已完成基础页面框架（首页、onboarding、farewell等），但缺少核心AI功能实现
- **技术限制**: 浏览器端运行，需考虑性能和兼容性；暂无后端服务，需使用前端可用的方案

## 功能需求

### FR-1: 多媒体文件上传系统
系统 SHALL 支持用户上传宠物照片（JPG/PNG/WebP）和视频文件（MP4/WebM），具备以下能力：
- 支持拖拽上传和点击选择两种方式
- 实时显示已上传文件的缩略图预览
- 文件格式和大小验证（单张图片≤10MB，单个视频≤50MB）
- 支持最少3张、最多10张照片的上传限制
- 显示上传进度和文件信息（名称、大小、类型）
- 支持删除已上传的文件

### FR-2: AI宠物形象生成引擎
系统 SHALL 能够基于上传的宠物照片生成2D卡通风格的宠物形象：
- 调用AI图像生成API（优先支持OpenAI DALL-E / Stable Diffusion API）
- 支持自定义生成参数：风格强度、色彩偏好、背景设置
- 显示生成进度状态（排队中、处理中、已完成、失败）
- 生成结果支持多种尺寸输出（512x512, 768x768, 1024x1024）
- 提供重新生成功能，支持微调提示词
- **降级方案**: 当API不可用时，提供基于CSS滤镜的卡通化效果作为备选

### FR-3: 2D动态宠物动画系统
系统 SHALL 为生成的宠物形象添加动态效果，使其看起来栩栩如生：
- **眨眼动画**: 随机间隔（3-8秒）触发的自然眨眼效果，持续150-300ms
- **呼吸动画**: 缓慢的上下浮动效果（周期4-6秒，幅度5-10px），模拟呼吸节奏
- **Idle动画**: 轻微的身体晃动或头部倾斜（随机触发，增加生动感）
- 动画参数可配置（速度、幅度、频率）
- 支持暂停/恢复动画播放
- 使用CSS Keyframes或Canvas实现高性能动画（60fps）

### FR-4: 桌面宠物模式界面
系统 SHALL 提供独立的桌面宠物展示窗口，具备以下特性：
- **始终置顶**: 窗口保持在其他应用之上（使用浏览器API或新窗口模式）
- **可拖拽移动**: 用户可自由拖动宠物到屏幕任意位置
- **点击交互**: 点击宠物时触发反馈动画（如跳跃、表情变化）
- **透明背景**: 宠物周围无背景框，融入桌面环境
- **大小调节**: 支持调整宠物显示尺寸（小/中/大）
- **快捷操作栏**: 提供最小化、关闭、设置按钮
- **双屏适配**: 支持在不同显示器间拖动

### FR-5: 完整用户工作流
系统 SHALL 提供从上传到展示的完整用户体验流程：
- **步骤1 - 媒体上传**: 引导用户上传宠物照片/视频，提供实时预览
- **步骤2 - 参数配置**: 选择生成风格、调整偏好设置
- **步骤3 - AI生成**: 调用AI服务，显示进度和预估时间
- **步骤4 - 结果预览**: 展示生成的静态图像和动态效果预览
- **步骤5 - 保存与应用**: 保存结果，启动桌面宠物模式
- 每个步骤都有明确的视觉反馈和错误处理

## 非功能需求

### NFR-1: 性能要求
- 页面首次加载时间 ≤ 3秒（在正常网络环境下）
- 图片上传和预览响应时间 < 500ms
- 动画帧率稳定在 ≥ 55fps（使用requestAnimationFrame优化）
- 内存占用合理（避免大量图片同时加载导致卡顿）

### NFR-2: 用户体验要求
- 响应式设计，支持桌面端（1920x1080及以上）和平板设备
- 操作流程直观，新手用户可在3分钟内完成首次生成
- 错误提示友好清晰，提供解决方案建议
- 支持中文界面，文案符合产品调性（温暖、治愈）

### NFR-3: 技术约束
- 纯前端实现为主，减少后端依赖
- 兼容现代浏览器（Chrome 90+, Firefox 88+, Safari 14+, Edge 90+）
- 代码遵循Next.js App Router规范
- 使用TypeScript或JavaScript（与现有代码保持一致）

## 技术假设
1. AI图像生成将使用第三方API（如OpenAI、Replicate、HuggingFace），需要用户提供API Key或使用演示模式
2. 桌面宠物模式将通过浏览器新窗口（window.open）+ 特定样式实现，而非原生桌面应用
3. 动画系统主要依赖CSS Animations和JavaScript定时器，不引入重型游戏引擎
4. 图片处理使用Canvas API进行客户端压缩和格式转换
5. 视频提取帧使用浏览器原生Video元素 + Canvas截图

## 验收标准

### AC-1: 图片/视频上传功能完整性
- **Given**: 用户访问AI宠物重建页面
- **When**: 用户通过拖拽或点击方式上传3-10张宠物照片（或1个视频文件）
- **Then**: 系统正确接收文件，显示缩略图预览，校验文件格式和大小，不符合要求的文件显示明确错误提示
- **Verification**: `programmatic`
- **测试要点**:
  - 上传JPG/PNG/WebP格式的图片文件成功
  - 上传MP4/WebM格式的视频文件成功
  - 上传超过10MB的图片时显示错误提示
  - 上传超过11张图片时提示数量超限
  - 删除已上传的文件功能正常
  - 拖拽上传区域高亮反馈正确

### AC-2: AI图像生成流程可用性
- **Given**: 用户已成功上传至少3张宠物照片并配置好生成参数
- **When**: 用户点击"开始生成"按钮
- **Then**: 系统调用AI生成API，实时显示处理进度（包括排队、处理、完成状态），最终返回生成的2D宠物图像
- **Verification**: `programmatic` (主流程) + `human-judgment` (生成质量)
- **测试要点**:
  - 正常情况下能在30秒内完成生成（取决于API响应速度）
  - 生成过程中显示加载动画和进度文字
  - API调用失败时显示友好的错误信息和重试选项
  - 降级方案（CSS滤镜卡通化）能够正常工作
  - 生成的图像风格符合预期（卡通化、保留宠物特征）

### AC-3: 动态宠物动画效果
- **Given**: 已生成静态宠物图像并启用动态效果
- **When**: 用户查看宠物展示区域
- **Then**: 宠物形象呈现自然的眨眼、呼吸、idle动画效果，动画流畅无卡顿
- **Verification**: `programmatic` (动画存在性) + `human-judgment` (视觉效果)
- **测试要点**:
  - 眨眼动画每3-8秒随机触发一次
  - 眨眼动作持续时间在150-300ms之间
  - 呼吸动画周期4-6秒，幅度5-10px
  - 动画帧率≥55fps（使用Performance API检测）
  - 点击"暂停动画"按钮后所有动画停止
  - 点击"恢复动画"按钮后动画重新开始

### AC-4: 桌面宠物模式交互
- **Given**: 用户已在预览页面看到满意的宠物形象
- **When**: 用户点击"启动桌面宠物"按钮
- **Then**: 系统打开新窗口/弹出层，显示可拖拽、可交互的桌面宠物，宠物始终置顶且背景透明
- **Verification**: `programmatic` (基础功能) + `human-judgment` (交互体验)
- **测试要点**:
  - 新窗口正确打开并显示宠物形象
  - 宠物可以通过鼠标拖拽到屏幕任意位置
  - 点击宠物时有明显的反馈动画
  - 宠物窗口始终在其他窗口之上
  - 背景透明，只显示宠物本身
  - 提供关闭按钮可以退出桌面模式
  - 支持调整宠物大小（至少3个尺寸档位）

### AC-5: 完整工作流连贯性
- **Given**: 新用户首次使用AI宠物重建功能
- **When**: 用户按照引导依次完成上传→配置→生成→预览→桌面展示的全部步骤
- **Then**: 整个流程顺畅无阻，每个步骤都有清晰的指引和反馈，总耗时控制在5分钟以内（不含AI生成等待时间）
- **Verification**: `human-judgment`
- **测试要点**:
  - 步骤指示器准确反映当前进度
  - 步骤间切换流畅，数据正确传递
  - 返回上一步时保留已填写的数据
  - 最终结果页面提供"重新生成"、"保存"、"桌面模式"等完整操作选项
  - 错误场景（网络中断、API失败）有合理的恢复机制
